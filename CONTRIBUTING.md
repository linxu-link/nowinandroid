# 如何成为贡献者并提交您的代码

## 贡献者许可协议

我们非常乐意接受您的示例应用和补丁！在接受之前，我们需要克服一些法律障碍。

请填写个人或公司贡献者许可协议（CLA）：

  * 如果您是编写原创源代码的个人，并且确定拥有知识产权，则需要签署[个人 CLA](https://developers.google.com/open-source/cla/individual)。
  * 如果您所在的公司希望允许您贡献工作，则需要签署[公司 CLA](https://developers.google.com/open-source/cla/corporate)。

请访问上述两个链接之一获取相应的 CLA 并了解如何签署和返回的说明。收到后，我们将能够接受您的拉取请求。

## 贡献补丁

1. 提交一个问题，描述您对相关仓库的建议更改。
2. 仓库所有者将及时回复您的问题。
3. 如果您的建议更改被接受，并且您还没有签署贡献者许可协议（见上文），请签署一份。
4. Fork 所需的仓库，开发并测试您的代码更改。
5. 确保您的代码符合您所贡献的示例中的现有样式。请参阅 [Google Cloud Platform 示例样式指南](https://github.com/GoogleCloudPlatform/Template/wiki/style.html) 了解该组织推荐的编码标准。
6. 确保您的代码有一套适当的单元测试，且全部通过。
7. 提交拉取请求。
